Chapter 12: Models of Computation

What is the purpose of a model?
Computer modeling allows scientists to conduct thousands of simulated experiments by computer.

List four phenomena of a model:
  1. It captures the essence, the important properties of the real thing.
  2. Probably differs in scale from the real thing.
  3. Omits some of the details of the real thing.
  4. Lacks the full functionality of the real thing.

List four things a computing agent should be able to do:
  1. Accept input
  2. Store information in memory and retrieve it from memory
  3. Take actions according to algorithm instructions
  4. Produce output

List four characteristics of an algorithm:
  1. Be a well-ordered collection
  2. Consist of unambiguous and effectively computable operations
  3. Halt in a finite amount of time
  4. Produce a result

What is the "halting problem" with regards to an uncomputable problem?
The problem Turing found is an ingenious one that itself involves Turing machine computations. A Turing machine that is executing an algorithm (a collection of Turing machine instructions) to solve some task must halt when begun on a tape containing input appropriate to that task.

Chapter 13: Simulation & Modeling
Define these terms:
  1. Scientific Method - The scientific method entails observing the behavior of a system and formulating a hypothesis that tries to understand and explain that behavior.
  2. Computational Model - also called simulation models. In a computer simulation, a physical system is not modeled as a small-scale replica but as a set of mathematical equations and/or algorithmic procedures that capture the fundamental properties and behaviors of a system.
  3. Discrete Event - A method that is relatively easy to understand and does not require a lot of complex mathematics. It is called discrete event simulation, and it is a very popular and widely used technique for building computational models.
  4. List and describe 7 reasons to construct a Simulation Model -
    1. Existence - The system might not yet exist, so it is not possible to experiment directly on the actual system.
    2. Physical Realization - The system is not constructed from entities that can be represented by physical objects.
    3. Safety - It might be too dangerous to experiment on the actual system or a physical replica.
    4. Speed of Construction - It might take too much time to construct a physical model.
    5. Time Scale - Some physical systems change too slowly or too quickly. For example, an elementary particle in a high-speed accelerator may decompose in 10215 seconds.
    6. Ethical Behavior - Some physical models have serious moral and ethi- cal consequences, perhaps the best known being the use of animals in medical research.
    7. Ease of Modification - If we are not happy with the design of a physical model, we need to construct a brand-new one.
