Computer Architecture Terms:
  1. RAM: is the hardware in a computing device where the operating system (OS), application programs and data in current use are kept so they can be quickly reached by the device's processor.
  2. Cache:cache is a high-speed data storage layer which stores a subset of data, typically transient in nature, so that future requests for that data are served up faster than is possible by accessing the data's primary storage location.
  3. Principle of locality: The high likelihood that when a program fetches a piece of data or an instruction:
    1. It will access that same instruction or piece of data in the very near future.
    2. It will soon access the instructions or data that are located near that piece of data, where “near” means an address whose numerical value is close to this one.
  4. ROM: a type of computer storage containing non-volatile, permanent data that, normally, can only be read, not written to.
  5. Memory cell size:Memory cells can be of different sizes: 8 bits, 16 bits, 32 bits, 64 bits.
  6. Memory Cell Address: a unique identifier used by a device or CPU for data tracking.
  7. Maximum Memory Size:the maximum amount of memory can be expanded within a system.
  8. Register:one of a small set of data holding places that are part of the computer processor.
  9. Fetch:the retrieval of data by a software program, script, or hardware device.
  10. Store:a document where information concerning the description code numbers of all items held in stock is kept for future use.
  11. Memory Access Time:how long it takes for a character in RAM to be transferred to or from the CPU.
  12. DASD:"Direct Access Storage Device," which is a type of storage device used in mainframe computers.
  13. Arithmetic And Logic Unit (ALU):a combinational digital circuit that performs arithmetic and bitwise operations on integer binary numbers.
  14. Control Unithandles all processor control signals.
Chapter 7:
Four things the interface between the Computer and the Hardware does:
  1. Hides from the user the messy details of the underlying hardware.
  2. Presents information about what is happening in a way that does not require in-depth knowledge of the internal structure of the system.
  3. Allows easy user access to the resources available on this computer.
  4. Prevents accidental or intentional damage to hardware, programs, and data.
Responsibilities of Systems Software:
  1. Hides the complex and unimportant (to the user) details of the internal structure of the Von Neumann architecture.
  2. Presents important information to the user in a way that is easy to understand.
  3. Allows the user to access machine resources in a simple and efficient way.
  4. Provides a secure and safe environment in which to operate.
Terms:
  1. Systems Software:a type of computer program that is designed to run a computer's hardware and application programs.
  2. User Interface :he point of human-computer interaction and communication in a device.
  3. Language Services:a type of service that enables communication between a brand and audiences outside of their language and culture.
  4. Memory Managers:the process of controlling and coordinating a computer's main memory.
  5. Information Managers:primarily concerned with the capture, digitization, representation, organization, transformation, and presentation of information.
  6. I/O Managers:user-provided objects that store asset and op outputs and load them as inputs to downstream assets and ops.
  7. Scheduler:the action of assigning resources to perform tasks.
  8. Utilities:A piece of code can be called a utility if it seems too small to be considered as a separate application, and too general-purpose to be considered as part of a particular program.
  9. Assembly Language:a type of low-level programming language that is intended to communicate directly with a computer's hardware.
Functions of an Operating System:
  1. The User Interface.
  2. System Security and Protection.
  3. Efficient Management of Resources.
  4. The Safe Use of Resources.
